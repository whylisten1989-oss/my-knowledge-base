# Excel 字段识别与映射

## 样本概况

- 文件：`数据参考722.xlsx`
- 工作表：`Sheet1`
- 使用范围：`A1:AI57`
- 字段数：35
- 数据行：56
- 汇总/平均行：2
- 人员候选行：54
- 有实际评价、询单或均响数据的候选行：9
- 账号与昵称不同：4 组
- 客服账号缺失：1 行（昵称“倪金”）
- 样本未发现“微信小店均响”或其他微信小店字段
- T 列表头为空，应忽略且不得导致列偏移

## 人员身份字段

| Excel 字段 | 标准字段 | 处理 |
|---|---|---|
| 客服账号 | `source_account` | 优先作为来源平台稳定身份键；不能直接当系统主键 |
| 客服昵称 | `source_nickname` / 展示名 | 用于显示和别名匹配 |

发现的账号/昵称差异：

- `训练师kk` → `小至`
- `kk` → `小梵`
- `小柒` → `小七`
- `运营助理刘梦丽` → `客服-小梦`

因此必须使用 `bi_agent_aliases` 保存来源账号、昵称与内部 `agent_id` 的映射。人员改名只新增/更新别名，不修改历史快照中的姓名。

“倪金”行缺少客服账号，首次导入时必须人工确认映射；未确认前不得自动写入正式 KPI。

## 核心 KPI 字段

| KPI | 优先原始字段 | 标准化结果 | 说明 |
|---|---|---|---|
| 满意率 | 有效好评数、有效差评数 | `satisfied_count`、`dissatisfied_count`、`satisfaction_rate` | 使用计数重新计算，不直接信任百分比 |
| 平均响应时长 | 工作时间平响时长 | `avg_response_seconds` | 已确认作为 V1 评分来源，统一解析为秒 |
| 转化率 | 询单人数、下单人数 | `inquiry_count`、`order_count`、`conversion_rate` | 使用人数重新计算，不建立客户订单系统 |

验证用字段：

- `有效评价数`：应与好评数 + 差评数一致；不一致时警告。
- `全天满意率`：与重新计算的满意率对比；不一致时警告。
- `询单转化率`：与下单人数 ÷ 询单人数对比；不一致时警告。

## 不进入核心 KPI 的字段

以下字段可以保留在导入行的 `raw_payload` 或 `ignored_payload` 中用于审计，但不写入核心 KPI 字段、不参与综合评分和排行榜：

- 客服在线天数、客服在线总时长、客服小休总时长；
- 最大接待人数、接待会话量、待回复会话量、已接待人数、正在接待人数、转出会话量、工作饱和度；
- 全天不满意率、工作时间不满意率、工作时间满意率；
- 全天平响时长（V1 不参与 KPI，只保留为非核心原始字段）；
- 工作时间首响时长、全天首响时长；
- 工作时间 3 分钟回复率、全天 3 分钟回复率；
- 服务问题会话数；
- 支付人数、退款人数、客服询单支付额、退款金额、退款后销售额；
- 空白 T 列；
- 未来出现的微信小店均响及任何表头包含“微信小店”的字段。

其中“下单人数”仅作为转化率分子，不扩展为客户订单、收入或 LTV 功能。

## 行过滤规则

解析阶段识别但不当作人员：

- `店铺汇总值`
- `店铺平均值`

Excel 中剩余 54 行全部进入“原始文件人员”集合，但没有任何人会因出现在 Excel 中而自动进入正式统计。

默认预选顺序：

1. 选择的常用名单中已启用且能匹配到 Excel 的人员；
2. 最近一次使用名单中仍有效的人员（仅在用户选择恢复时使用）；
3. 新人员默认不勾选，并显示“新人员”提示；
4. 停用人员默认不勾选，并显示“已停用”提示；
5. 无关人员默认不勾选；
6. 历史名单中存在但本次 Excel 缺失的人员显示“本次缺失”，不记未达标。

## 日期与团队字段缺口

本样本没有明确的业务日期列，也没有团队/平台列。

阶段一默认：

- 业务日期由文件名尝试识别（如 `722`）后让用户确认；不能仅凭文件名直接入库。
- 团队/平台来自用户选中的统计名单、人员主数据或导入范围选择。
- 若未来 Excel 增加团队/平台列，则仅用于匹配和校验，不自动覆盖人员主数据。

## 微信小店字段兼容规则

解析器应先按表头名称映射，而不是依赖固定列序号。任何表头符合以下模式时直接进入忽略集合：

- 精确等于 `微信小店均响`
- 包含 `微信小店`
- 管理员配置的其他忽略字段

忽略字段不会触发“缺少必填列”，不会改变其他字段索引，也不会进入校验、KPI、排名、趋势、Dashboard 或数据库核心字段。
