# 客服人力运营中心｜仓库投放说明

## 推荐放置方式

请不要覆盖仓库根目录现有的 `index.html`。

将压缩包中的整个 `workforce-center` 文件夹复制到仓库根目录，最终结构应为：

```text
my-knowledge-base/
├─ index.html
├─ customer-bi.html
├─ css/
├─ js/
└─ workforce-center/
   ├─ index.html
   ├─ css/
   │  └─ workforce-center.css
   └─ js/
      ├─ mock-data.js
      └─ workforce-center.js
```

提交到 `main` 后，等待 Vercel 部署完成，访问：

```text
https://你的域名/workforce-center/
```

备用地址：

```text
https://你的域名/workforce-center/index.html
```

## 本版本已处理

- 与仓库现有首页、客服 BI 页面完全隔离，不覆盖同名文件。
- 日视图和周视图使用真实日期字段。
- 日期、小组、人员、班次、状态和熟练度筛选可用。
- 当前时间线会按分钟更新，不会刷新整个时间轴。
- 有效工时会扣除重叠的休息、培训、会议和请假时段。
- Excel/CSV 可包含多个日期，不会全部混在同一天展示。
- 导出只导出当前日期和当前筛选范围。
- 上传模板由浏览器动态生成，不依赖会被 `.vercelignore` 排除的 CSV 文件。
- 移除 Google Fonts 依赖，减少国内网络加载失败。
- Excel 解析库与现有 BI 页面统一使用 SheetJS 0.20.3。
- 上传后的排班数据保存在当前浏览器本地，刷新页面后仍可继续查看。

## 当前数据保存方式

上传的 Excel/CSV 在浏览器本地解析，不会自动上传到 GitHub、Vercel 或 Supabase。

数据保存在当前浏览器的 LocalStorage：

- 同一台电脑、同一浏览器刷新后仍会保留。
- 更换电脑、浏览器或清除浏览器数据后不会保留。
- 后续需要多人共享同一份排班时，再接入 Supabase。

## 注意事项

- `.xlsx/.xls` 解析需要能够加载 SheetJS CDN；网络异常时可以先把文件另存为 CSV 上传。
- 当前属于第一阶段排班调度样板，其他导航暂时显示“建设中”。
- 暂未修改现有 `index.html` 导航，先通过独立网址访问。
